≈ $50/mo — about $600/yr of eliminable waste, zero performance impact.

Honest calibration. This environment was deliberately waste-heavy and its compute runs on free-tier instances, so the percentage reduction is unusually high. On real production accounts — where compute dominates and isn't free — the same methodology typically yields a 25–45% total-bill reduction. Cost work that inflates its numbers doesn't survive a second look, so the numbers here are stated plainly.

The audit — seven findings

  • F1NAT gateway carrying endpoint-eligible traffic (hero)Egress is S3/DynamoDB-bound, and Amazon Linux 2023 repos are S3-hosted — so even OS updates don't need a NAT. Replaced ~$33/mo of NAT with free S3 + DynamoDB gateway endpoints.
  • F2Idle Elastic IPsTwo unassociated EIPs billing for nothing → released.
  • F3Unattached EBS + gp2Two orphaned 50 GB volumes deleted; RDS storage migrated gp2 → gp3 (~20% cheaper, better baseline IOPS).
  • F4No S3 lifecycle policyAdded Standard → IA (30d) → Glacier (90d) → expire (365d). Prevention finding — the biggest real-account saving once data scales.
  • F5Logs never expireCloudWatch retention set to 30 days, capping unbounded log growth.
  • F6No tagging governanceFull tag set enforced via provider default_tags; AWS Config required-tags rule recommended for ongoing enforcement.
  • F7Always-on compute (modeled)An off-hours scheduler running a fleet 60h/wk vs 168h/wk is a ~64% compute cut — documented with the math rather than built, since free-tier demo instances make its dollar impact zero here.

Evidence — measured, not assumed

The account is an AWS Free Plan account, so actual billing is $0. The savings above are AWS published on-demand pricing applied to measured usage — these are the real numbers the environment generated.

Method

A real engagement structure: discovery (Cost Explorer drivers, usage-type breakdown, anti-pattern hunt) → recommendations grouped by effort and risk (quick wins → medium → strategic) → implementation as a Terraform diff with a measured before/after, and a full teardown when done. Every figure here is reproducible from the repository.

Scope & honesty. A simulated environment modeled on a typical small-SaaS workload — not a real client. The method, the Terraform, the measured usage, and the boot logs are genuine; the "client" is a stand-in. Trustworthy cost work starts with not overstating the result.
